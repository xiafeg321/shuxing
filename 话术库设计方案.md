# 数星 - 话术库设计方案

## 一、整体架构

```
用户消息 → 语境匹配引擎 → 判断层级 →
   ├─ 0级（简单情绪/问候）→ 话术库直接匹配
   ├─ 1级（中等情境）     → 话术库 + 个性填充
   └─ 2级（深度倾诉）     → 转大模型
```

## 三、语境匹配增强设计

### 3.1 多层匹配规则

```
第一层：关键词匹配（现有）
  例：含"难过" → 匹配A1

第二层：组合关键词匹配（新增）
  例：含"分手"+"怎么办" → A1+A5 综合响应

第三层：否定词检测（新增）
  例："我不难过" → 有"不"+"难过" → 不匹配A1
  例："没那么伤心" → 有"没"+"没那"+"伤心" → 降低强度

第四层：上下文参考（新增）
  例：用户前一条说"好难受"（A1），现在说"睡不着"（A4）
  → 优先判断A1+A4混合
```

### 3.2 强度分级

每条匹配结果附带强度值：

| 信号 | 强度 | 示例 |
|------|:----:|------|
| 单个关键词 | 1 | "难过" |
| 多个同类型关键词 | 2 | "太难过了想哭" |
| 感叹/重复 | +1 | "真的真的好难受！" |
| 否定词 | -1 | "其实还好" |
| 长度超过20字 | +1 | 详细描述 |

- 强度≤1 → 话术库简单回复
- 强度2 → 话术库加强回复
- 强度≥3 → 转大模型

## 五、混合策略判断逻辑

```
用户输入 → 语境匹配引擎

情境判断：
  1. 关键词匹配（15类）
  2. 强度计算（1-3级）
  3. 否定词修正
  4. 上下文参考（最近3轮）
  5. 对话阶段判断（L1/L2/L3/L4）

路由决策：
  ┌ 强度1 + 简单场景 → 话术库（0级）：直接匹配话术
  ├ 强度2 + 有上下文 → 话术库+填充（1级）：话术+个性化
  ├ 强度3+ → 大模型（2级）：完全AI生成
  ├ 特殊场景（求助/分析）→ 大模型（2级）
  └ 不确定 → 默认走话术库（安全降级）

话术匹配细化：
  匹配到类型后 →
    子类型判断（如A1下分轻/重/混合）
    → 结合对话阶段调整句式
    → 结合MBTI调整长度/风格
    → 替换{n}（区分：昵称vs称呼）
```

## 六、昵称区分方案

**当前bug：** nickname 只有一个值（对方名字），但 {n} 有些语境应该是"你自己"

**解决方案：**
```
const userName = cm?.L1?.userName || ''     // 用户自己的名字/称呼
const partnerName = cm?.L1?.nickname || ''   // 对方的名字（原来的nickname）

替换规则：
  {n} → 大部分场景 = partnerName（对方名字）
  {u} → 新增占位符 = userName（用户称呼）
```

**具体话术例子（写信风格区分）：**
- `{n}，想哭就哭吧` → 对方名字（"小雨，想哭就哭吧"）
- `好起来的就是{u}呀` → 用户名字（"好起来的就是峰哥呀"）

## 七、渐进式对话阶段（各阶段话术风格）

| 阶段 | 名称 | 风格 | 话术侧重 |
|:----:|------|------|---------|
| L1 | 破冰收集 | 温柔好奇 | 了解情况、收集信息 |
| L2 | 建立信任 | 温暖接纳 | 建立信任感、确认感受 |
| L3 | 深化连接 | 信任陪伴 | 更深层的对话、回忆共鸣 |
| L4 | 稳固陪伴 | 坚定在我 | 稳定的长期陪伴关系 |
