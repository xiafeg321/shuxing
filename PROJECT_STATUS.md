## 💡 技术决策记录

### 架构决策
1. **纯前端架构**
   - 优点: 部署简单，成本低，隐私保护
   - 缺点: 功能受限，无法复杂计算
   - 决策理由: 快速验证核心需求，降低技术复杂度

2. **规则引擎而非AI模型**
   - 优点: 零成本，可控性强，响应快
   - 缺点: 灵活性有限，需要手动维护
   - 决策理由: 第一阶段验证用户需求，后续可升级

3. **本地存储而非云端**
   - 优点: 隐私保护，离线可用，响应快
   - 缺点: 数据不跨设备同步
   - 决策理由: 情感话题敏感，隐私优先

### 技术选型
1. **无框架原生开发**
   - 理由: 学习成本低，部署简单，性能好
2. **localStorage存储**
   - 理由: 简单可靠，浏览器原生支持
3. **响应式CSS**
   - 理由: 移动端优先，覆盖所有设备

## 🔧 故障恢复指南

### 常见问题解决
1. **页面无法加载**
   ```
   检查: 服务器是否运行 (python3 -m http.server 3000)
   解决: 重启服务器，检查端口占用
   ```

2. **功能不正常**
   ```
   检查: 浏览器控制台错误信息
   解决: 访问 debug-test.html 进行诊断
   ```

3. **数据丢失**
   ```
   检查: 浏览器localStorage是否被清空
   解决: 重新设置人格模型，对话历史无法恢复
   ```

### 紧急恢复步骤
1. **备份现有代码**: `git add . && git commit -m "紧急备份"`
2. **检查服务器**: 确保HTTP服务器运行
3. **诊断问题**: 使用测试页面定位问题
4. **回滚更改**: `git checkout -- .` 恢复文件
5. **重新部署**: 重启服务器测试
